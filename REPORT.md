## Project overview
- Language: Python 3.x (all scripts and backend services).
- Purpose: Given a `case_fact` (free text), predict:
  - Verdict (multi-class; e.g., `Guilty` / `Not Guilty`)
  - IPC section (multi-class)
  - Relevant laws (multi-label)
  - Penalty in months (regression)
  - Provide similar past cases using dense retrieval (SBERT + NearestNeighbors)

## Detailed architecture
- Core model: Multi-task Transformer with a shared DistilBERT backbone and four heads (training and inference code in `train.py`, `inference.py`, `app.py`):
  - Backbone: `DistilBertModel` (`distilbert-base-uncased`) or `AutoModel.from_pretrained(MODEL_NAME)` where `MODEL_NAME` is `distilbert-base-uncased`.
  - Hidden dimension: backbone `config.hidden_size` (DistilBERT default: 768).
  - Dropout: 0.2 applied to pooled CLS embedding.
  - Heads:
    - Verdict head: Linear(h, V) where V = number of verdict classes. Uses CrossEntropyLoss.
    - IPC head: Linear(h, I) where I = number of IPC classes. Uses CrossEntropyLoss.
    - Laws head: Linear(h, L) where L = number of law labels. Uses BCEWithLogitsLoss with `pos_weight` to counter class imbalance.
    - Penalty head: Linear(h, 1) predicting a scaled penalty value; training uses MSELoss and a `MinMaxScaler` is used to scale/inverse-scale penalty values.

Loss composition used in training:
- total_loss = loss_verdict + loss_ipc + LAW_LOSS_WEIGHT * loss_laws + loss_penalty
- `LAW_LOSS_WEIGHT` is configurable (default 2.0 in `train.py`) to emphasize multi-label learning for laws.

Multi-label thresholding:
- During inference `app.py` uses sigmoid then threshold `>= 0.5` to select relevant laws.
- During validation in `train.py` a lower threshold (0.25) is used for reporting F1 scores (developer choice to be more permissive in metrics).

Retrieval stack:
- SBERT model: `all-MiniLM-L6-v2` used to create dense embeddings for `case_fact` (via `sentence_transformers.SentenceTransformer`).
- Embedding storage: `case_embeddings.npy`, metadata arrays `case_ids.npy`, `case_summaries.npy`.
- NearestNeighbors: scikit-learn `NearestNeighbors(metric='cosine', n_neighbors=4)` is fitted on embeddings and saved to `nearest_neighbors.joblib`.

## Training configuration & hyperparameters (from `train.py`)
- Dataset: `balanced_final_dataset_v6.csv` (generated by `dataset_generator.py`).
- Split: train/val split = 85% / 15% (stratified by verdict label).
- Batch size: 8
- Epochs: 4
- Max token length: 256
- Learning rate: 3e-5
- Warmup percentage: 0.06 (linear warmup scheduler with `get_linear_schedule_with_warmup`).
- Optimizer: `AdamW`
- Mixed precision: Automatic (AMP) enabled if `device.type == 'cuda'`.
- Random seed: 42
- DataLoader: `num_workers=0` and `pin_memory=True` (Windows-safe settings in `train.py`).
- Losses: CrossEntropyLoss (verdict & IPC), BCEWithLogitsLoss (laws, with computed `pos_weight`), MSELoss (penalty).

How `pos_weight` is computed:
- pos_weight = (neg_counts + 1) / (pos_counts + 1)
  - Computed per-label using the MultiLabelBinarizer matrix to balance rare labels.

Model checkpointing:
- The training script saves the best model by Verdict accuracy to `model_artifacts/model_state.pt` and also saves tokenizers and encoders there.

## Data & preprocessing details
- Dataset columns expected:
  - `case_id` (unique id),
  - `case_fact` (text input),
  - `verdict` (label),
  - `ipc_section` (label),
  - `relevant_laws` (stringified list or CSV of labels),
  - `penalty_months` (numeric),
  - `similar_case_summaries` (concatenated summary strings for retrieval display),
  - other metadata like `severity`.
- Parsing `relevant_laws`: `train.py` implements `parse_laws` that accepts either a Python list string (e.g. "['IPC1','IPC2']") or comma-separated values and always converts to a Python list.
- Preprocessing (`preprocess.py` and local `preprocess` in `train.py`):
  - Ensure value is str, convert to lowercase, remove `\n` / `\r`, and normalize whitespace with `" ".join(text.split())`.
- Tokenization: uses Hugging Face `AutoTokenizer` / `DistilBertTokenizer` with `padding='max_length'`, `truncation=True`, and `max_length=MAX_LEN`.

## Artifacts inventory (exact files and purpose in `model_artifacts/`)
- `model_state.pt` — PyTorch state dict of the saved multi-task model.
- `le_verdict.pkl` / `le_ipc.pkl` — LabelEncoder objects (map label ↔ integer) saved via `joblib`.
- `mlb_laws.pkl` — MultiLabelBinarizer fitted on `relevant_laws` classes.
- `scaler_penalty.pkl` — MinMaxScaler used to scale penalty months during training.
- `case_embeddings.npy` — SBERT dense embeddings for all cases.
- `case_ids.npy` — Ordered case IDs aligned with `case_embeddings.npy`.
- `case_summaries.npy` — Similar-case summaries used for display.
- `nearest_neighbors.joblib` — Fitted `NearestNeighbors` object for retrieval (cosine metric).
- `tfidf_vectorizer.joblib`, `clf_verdict.joblib`, `clf_ipc.joblib`, `clf_laws.joblib` — saved classical ML artifacts (experimental/baseline models or previous versions).
- Tokenizer artifacts: `tokenizer.json`, `vocab.txt`, `tokenizer_config.json` — saved tokenizer files for deterministic tokenization and for `from_pretrained` loading from local folder.

## Inference & API behavior
1. `app.py` provides a FastAPI endpoint `POST /predict` which expects JSON `{"case_text": "..."}` and returns a JSON object with keys:
   - `verdict` (string),
   - `verdict_confidence` (float),
   - `ipc_section` (string or `-`),
   - `ipc_confidence` (float or `None`),
   - `relevant_laws` (list of strings),
   - `penalty_months` (float)

2. `inference.py` provides a richer offline routine `predict_case(text)` that also returns `similar_cases` (list of strings) and a short textual `case_summary` assembled from predictions and similar case summaries.

3. Key inference behavior notes:
  - If predicted verdict (decoded via `le_verdict.inverse_transform`) is `Not Guilty`, the API short-circuits and returns no IPC/laws/penalty.
  - Laws: `app.py` uses `sigmoid(logits) >= 0.5` to select multi-labels. If no label passes, the top-3 highest-probability labels are returned.
  - Penalty: the model outputs a scaled value, which is inverse-transformed by `scaler_penalty` to return penalty months. `app.py` rounds to two decimals.

## Backend, auth, and DB
- The folder `backend/` contains a small FastAPI app:
  - `main.py` serves the frontend static files under `/` and mounts the `frontend/` directory at `/static`.
  - `auth.py` exposes signup and login endpoints under `/auth` using OAuth2 password flow and dependency injection.
  - `security.py` uses `jose.jwt` for `HS256` JWT tokens and `passlib` (bcrypt) for password hashing.
  - `database.py` configures a local SQLite DB at `sqlite:///./users.db` and provides `SessionLocal` and `Base` for models.

Auth flow summary:
- `POST /auth/signup` accepts `UserCreate` (name, email, password) and stores the user with hashed password.
- `POST /auth/login` accepts `UserLogin` and returns an access token JWT on success.
- `get_current_user` helper decodes JWT and loads user from DB.

Security note: The file `backend/security.py` currently contains a hardcoded `SECRET_KEY`. Replace this with a strong secret stored in environment variables for production.

## How to run (development + training + inference)

1) Run the frontend + lightweight backend (serves static UI):

```powershell
uvicorn backend.main:app --reload
# open http://127.0.0.1:8000 in a browser (index.html served)
```

2) Run the model API directly (the `app.py` which contains the `/predict` endpoint):

```powershell
python app.py
# or run via uvicorn: uvicorn app:app --reload
```

3) Call the `/predict` endpoint (example):

```powershell
curl -X POST "http://127.0.0.1:8000/predict" -H "Content-Type: application/json" -d '{"case_text": "The accused ..."}'
```

4) Train the model (on GPU recommended):

```powershell
python train.py
```

Training will save artifacts into `model_artifacts/` including model state, encoders and SBERT embeddings.

5) Offline inference example (using `inference.py`):

```powershell
python inference.py
# or import predict_case from inference.py from another script
```
